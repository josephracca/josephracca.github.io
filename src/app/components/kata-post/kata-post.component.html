What the heck is a permutation?

I should talk a bit about what a permutation is. So from my understanding, a permutation is any combination or rearrangement of characters in a string. In this case, the possible number of permutations is calculated by taking the total count of characters, or string length, and running a factorial on it. So for a string of length 6, the number of possibly combinations, would be 6! = 6 × 5 × 4 × 3 × 2 x 1, which is the equivalent of 720. 720 total combinations. This is calculated as follows: the first letter could be any one of the six letters in the string, then once one letter is used, that leaves 5, then 4 and so on, until you have no more letters to use in the combination.

Now that we have that out of the way, let's move on! (Again, we don't have to worry about duplicate permutations here because all characters are unique; IE, in "aba", "aba" === "aba" (see below), which would essentially change the way we calculate the number of total unique permutations.) 

        			
        			(string[0])+(string[1])+(string[2]) === (string[2])+(string[1])+(string[0])

At this point, it was becoming very clear to me what was going on. The pattern was starting to make sense, but I had to check to see if there was a difference between the strings with an even length and those with an odd length, and based on the sketches, there was a difference. Even-length strings had one middle character that separated the front half of the original string from the back half, whereas odd-length strings had two (with the exception of string length of 3, figure 5).

It had also become apparent that the string was being divided into three parts to get to the final result: a "front", "middle", and "back". In the middle permutation, the order would become MBF, with each of those segments being filled in reverse order. (See example of 10-length string, figure 8)

The middle permutation for "abcdefghij" is "ejihgfdcba".

Spaced for clearer visualization: " abcd e fghij " becomes "e jihgf dcba".

So breaking it down a bit further, abcd gets reversed, then takes the last position, e remains a single char, and takes the front/first position, and fghij gets reversed and takes the middle position.

So this is all great in layperson terms, but what does this look like in code? Well here it is in javascript. I'll break it down a bit further with some classic comments.

The Solution:

function middlePermutation(s) {
  let start, mid, end, midway = "";

  let revArr = s.split("").sort().reverse();
  
  let halfE = s.length/2;
  let floorHalf = Math.floor(halfE);
  let stopper = (s.length%2 !== 0 ) ? floorHalf+2 : halfE+1;
  let midStart = (s.length%2 !== 0 ) ? floorHalf : halfE;
  
  start = revArr.slice(midStart, stopper).join("");
  mid = revArr.slice(0, midStart).join("");
  end = revArr.slice(stopper, s.length).join("");
  
  midway = start + mid + end;

  return midway;
}

Breaking down the solution:

// function declaration, middlePermutations is our function name and it accepts one argument through its parameter "s"

function middlePermutation(s) {

//declare some variables to store our temp arrays/strings we create from our original string
// start = the front end of our string
// mid = the middle char or pair of chars that act as our splitter
// end = the back end of our string
// midway = our awesome middle permutation
// I initialize all of these to empty strings

  let start, mid, end, midway = "";

// revArr is a variable that holds our temporary array, which is splitting our string, sorting it alphabetically, then reversing it, resulting in array of chars, in reverse alphabetical order

  let revArr = s.split("").sort().reverse();
  
//now we need to determine how to handle the string based on the length, ie odd or even

// halfE gives us the length divided in half but we need to floor it in case we have a decimal (from our odd strings), these variables will determine our starting points and stopping points for our start, mid, and end.

  let halfE = s.length/2;
  let floorHalf = Math.floor(halfE);

//checks odd or even and provides the correct stopper and midStart point

  let stopper = (s.length%2 !== 0 ) ? floorHalf+2 : halfE+1;
  let midStart = (s.length%2 !== 0 ) ? floorHalf : halfE;

//once those are claculated, we can use those numbers in our slice method on the reversed array, then rejoin those arrays back into strings
  
  start = revArr.slice(midStart, stopper).join("");
  mid = revArr.slice(0, midStart).join("");
  end = revArr.slice(stopper, s.length).join("");

//finally we rebuild our string by concatenating these resulting strings
  
  midway = start + mid + end;

//then we can return this resulting string

  return midway;
}
